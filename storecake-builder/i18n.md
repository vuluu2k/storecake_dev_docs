# Internationalization (i18n)

`builderx_spa` dùng **vue-i18n 9** + **i18n-ally** (VSCode) để quản lý chuỗi đa ngôn ngữ.

## Cấu trúc

```
src/i18n/
├── index.js                # createI18n, autoload locale
├── locales/
│   ├── vi.json
│   ├── en.json
│   ├── th.json
│   └── …
└── helpers.js              # (nếu có) helper format custom
```

Locale mặc định: `vi`. Fallback: `en`.

## Cấu hình i18n-ally

`.vscode/settings.json` đã có sẵn:

```json
"i18n-ally.localesPaths": ["src/i18n/locales"],
"i18n-ally.keystyle": "nested",
"i18n-ally.sourceLanguage": "vi",
"i18n-ally.sortKeys": true,
"i18n-ally.translate.engines": ["google"],
"i18n-ally.regex.usageMatchAppend": [
  "t\\.success\\(\\s*['\"]({key})['\"]",
  "t\\.error\\(\\s*['\"]({key})['\"]"
]
```

→ Nhờ regex append, các helper `t.success('...')`, `t.error('...')` cũng được detect.

## Cách dùng

### Options API

```vue
<template>
  <a-button>{{ $t('product.create_new') }}</a-button>
</template>

<script>
export default {
  methods: {
    onCreate() {
      this.$t('product.create_new');
      this.$message.success(this.$t('product.created'));
    },
  },
};
</script>
```

### Composition API

```vue
<script setup>
import { useI18n } from 'vue-i18n';

const { t, locale } = useI18n();
const label = t('product.create_new');
</script>
```

### Trong store / module thuần

```js
import i18n from '@/i18n';
i18n.global.t('product.create_new');
```

## Quy tắc đặt key

* Nested theo feature: `product.create_new`, `order.status.completed`, …
* **Không** dùng key trùng cho 2 ngữ cảnh khác nhau.
* Đặt key tiếng Anh thuần (snake_case), nội dung dịch theo locale.
* Khi xoá field, **xoá luôn** key ở mọi locale (tránh stale).
* Locale `vi` là source language → dịch tự động Google cho locale khác qua i18n-ally.

## Lint

`eslint-plugin-i18n-json` đã được bật cho `src/i18n/locales/*.json`:

```json
// package.json
"lint-staged": {
  "src/i18n/locales/*.json": "eslint --fix",
  "*.{js,ts,vue}": "eslint --fix"
}
```

Plugin sẽ:

* Sort key đồng nhất giữa locale.
* Cảnh báo placeholder `{name}` không match giữa các locale.
* Báo missing key (đối chiếu với `sourceLanguage`).

## Thêm locale mới

1. Tạo file `src/i18n/locales/<code>.json` (copy từ `vi.json`).
2. Mở i18n-ally trong VSCode → "Translate missing keys" để generate nháp.
3. Đăng ký locale vào `src/i18n/index.js` (`messages` map) + bổ sung trong UI chọn locale (`useLocaleStore`).
4. Cập nhật danh sách locale ở backend (`builderx_api`) nếu cần.

## Pitfalls

* Đừng inject biến vào key (`$t(\`product.${type}\`)`) khi `type` không cố định → i18n-ally không detect, dễ rớt khi cleanup.
* Format số/ngày: dùng `$n`, `$d` với cấu hình tại `src/i18n/index.js` (numberFormats / datetimeFormats) thay vì chuỗi cứng.
* Khi load lazy locale (split chunk), nhớ `i18n.global.setLocaleMessage(locale, msgs)` rồi `setLocale(locale)`.
